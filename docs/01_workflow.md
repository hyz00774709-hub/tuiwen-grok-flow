# 一键流程 V0.1

## 1. 输入

用户可以提供三类输入：

- 小说名字 + 平台/链接
- 小说免费章节文本
- 已授权全文或章节文本

每本书都可以额外提供一条参考视频，用于提取视觉风格、字幕风格、UI 类型和节奏。

## 2. 内容提取

如果用户只给书名，流程只提取公开可访问的信息和免费章节。遇到 VIP、登录、付费墙时停止，不做绕过。

输出：

- 书名、作者、平台、链接
- 免费章节范围
- 核心设定
- 主角目标
- 主线冲突
- 前 3 秒爆点候选
- 前 60-90 秒测试片段候选

## 3. 原创解说重构

目标不是复述原文，而是改造成推文解说稿。

要求：

- 保留主线、人物关系、核心爽点
- 改写叙述顺序和句式
- 爆点前置
- 加强悬念、危机、反转、系统爽点
- 删除低效铺垫
- 避免连续复用原文表达

## 4. 10 秒段落切分

当前默认每段 10 秒。每段需要有一个清楚的画面任务，不能只有旁白。

输出：

- Segment ID
- Time Range
- Narration
- Story Function
- Visual Task
- UI Overlay
- Generation Mode

## 5. 分镜控制卡

分镜控制卡是后台中控协议，不需要给普通用户复杂展示。

它负责把旁白、画面、UI、人物、场景和生成方式对齐。

## 6. 生成提示词

根据分镜控制卡生成两类提示词：

- GPT Image 2 出图提示词
- Grok 10 秒视频提示词

系统面板、排行榜、字幕、属性卡等中文文字，优先由代码模板叠加。

## 7. 合成与审核

第一版目标是生成可审核草稿：

- 旁白音频
- SRT 字幕
- AI 背景画面
- UI 模板叠加
- 音效建议
- 预览 MP4 或剪映素材包

审核通过后，再批量扩展。
